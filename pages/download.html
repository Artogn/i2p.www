Clean installs

  • GUI installer (requires Swing):
    install.jar (SHA1 a85ad25d4fa5ce4c914fb287bde0605e97a3c1a6)
    Right-click "save as", then either double click on it or run java -jar install.jar
  • Headless install:
    i2p.tar.bz2 (SHA1 75b97e12dde6f0d4061944b1ee34d06975d936d7)
    Run (tar xjvf i2p.tar.bz2 ; cd i2p ; vi install-headless.txt)
  • Source install:
    i2p_0_4_0_1.tar.bz2 (SHA1 58c822e45662c94dc3cfff1d91fb84e1cb457233)
    Alternately, you can fetch the source from cvs (CVS HEAD gets you the most recent updates, while the tag i2p_0_4_0_1 gets you the release)
    Run (tar xjvf i2p_0_4_0_1.tar.bz2 ; cd i2p_0_4_0_1 ; ant dist) then either run the GUI installer or headless install as above
  • After running one of the above installs, you will want to go to the router console and follow the instructions (be sure to enter your IP address / domain name, check any NAT/firewall, and reseed)
Important: if you had a 0.3 install running in the last day, you must use the update procedure

Updates from 0.4:

  1. If you previously set I2P to run as a windows service, we need to temporarily remove that. Go to http://127.0.0.1:7657/configservice.jsp and select "Don't run I2P on startup".
  2. Go to http://127.0.0.1:7657/configservice.jsp and select "Shutdown gracefully"
  3. Download http://dev.i2p.net/i2p/i2pupdate.zip (SHA1 1376d3410c49006ce55f51ebe4d496b55ec90762) to your I2P installation directory and extract it there (it will overwrite files such as lib/i2p.jar and i2prouter.bat).
    If you have modified your wrapper.conf, please merge that with the new wrapper.config
  4. Go grab a beer (aka wait 10 minutes - there was a bug in the 0.4 graceful shutdown code)
  5. If you removed the windows service in step 1 above, lets add it back in - run install_i2p_service_winnt.bat
  6. Start the router back up
    • on *nix: sh i2prouter start
    • on windows: Start-->Run-->net start i2p
    • on windows 9x: click on i2prouter-win9x.bat (make sure "javaw" is in your path)

Updates from 0.3.1.1-0.3.4.3:

  • If you can use a different external port (the old default was "8887"), simply do a clean install into a brand new directory.
  • If you cannot use a different external port:
    • Stop the old router
    • Do a clean install into a brand new directory
    • Stop the router gracefully
    • Copy the following files from the old directory into the new one:
      • router.info
      • router.keys
      • sam.keys
      • keyBackup/*
    • Go to http://localhost:7657/config.jsp and tell it to use your old external host and port number
  • If you were running any I2PTunnel servers, please see the migration guide
