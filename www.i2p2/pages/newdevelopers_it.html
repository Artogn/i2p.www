  • Non siete pronti ancora per lavorare sul codice? Provate, prima a getting involved.
  • Per iniziare a sviluppare i2p router o applicazioni integrate, imparate il Java, amate il Java :)
  • Leggete l'introduzione del funzionamento, anche altri documenti, anche introduzione tecnica, e tutti gli altri documenti associati.
  • Per iniziare a sviluppare i2p router o applicazioni integrate, procuratevi la repository sorgente dal server monotone - Istruzioni:
    • Installate monotone (la pagina Ã¨ in inglese)
    • Abilitate un tunnel, tramite i2ptunnel, alla porta 8998 che punta all'indirizzo mtn.i2p2.i2p
    • tramite la console scrivete: mtn -d i2p.mtn db init
    • tramite la console scrivete: mtn -d i2p.mtn pull 127.0.0.1:8998 i2p.i2p
    • tramite la console scrivete: mtn -d i2p.mtn co --branch=i2p.i2p
    Accesso non-anonimo: mtn.i2p2.de:4691 o mtn.i2p-projekt.de:4691
    "Ramo" del sito: usate i2p.www al posto di i2p.i2p.
    Lista completa dei rami: controllate questo indirizzo.
    Versione completa della guida: controllate questa pagina.
  • Per compliare il codice, avete bisogno del Sun Java Development Kit 5 o maggiore, o un JDK equivalente (Sun JDK 6 fortemente raccomandato) e Apache ant versione 1.7.0 o maggiore.
  • Per creare o lavorare sulle traduzione della console, avete bisogno di xgettext, msgfmt, e msgmerge tools, che potete prelevare da questa pagina.
  • Per sviluppare nuove applicazioni, leggete laguida su come creare nuove applicazioni.
  • Guardate zzz's TODO lists, TODO list di questo sito o Trac per idee.
  • Salutate gli sviluppatori nel canale irc #i2p
  • Controllate il seguente link per conoscere i requisit per ricevere i privilegi di commit.
  • Versione corta su come creare le chiavi che avete intenzioni di usare per eseguire i commit:
    • mtn genkey yourname-transport@mail.i2p (non usate nessuna passphrase)
    • mtn genkey yourname@mail.i2p (usate una passphrase)
    • mtn pubkey yourname-transport@mail.i2p (send this to a mtn repo operator to get push privileges)
    • mtn pubkey yourname@mail.i2p (inviatela a un release manager per ricevere i privilegi di commit)
    • mtn ci -k yourname@mail.i2p (check in usando questa chiave)
    • mtn sync -k yourname-transport@mail.i2p (per eseguire il push/pull/sync usate questa chiave)
